# Photo Standard

Статус: Draft  
Версия: 0.1  
Проект: Soft ICE Platform / «У Тимоши»

## 1. Назначение

Документ фиксирует стандарт подготовки изображений для интерфейсов, сайта, терминала, Mini App, CRM, презентаций и рекламных материалов.

Цель: все изображения должны выглядеть как часть единой продуктовой системы, а не как набор случайных картинок.

## 2. Каналы использования

Изображения используются в:

- Telegram Mini App;
- MAX Mini App;
- Web App;
- терминале вендингового аппарата;
- сайте `utimoshi.ru`;
- CRM;
- Seller App;
- социальных сетях;
- рекламных материалах;
- презентациях;
- документации для партнёров.

## 3. Главный принцип

> Новое изображение добавляется в проект только если оно соответствует единому стилю, формату, ракурсу, свету и структуре хранения.

## 4. Базовая структура медиатеки

```text
frontend/miniapp/public/images/
├── icecream/
│   ├── base/
│   ├── syrup/
│   ├── topping/
│   └── composed/
├── syrups/
├── toppings/
├── club/
├── branding/
│   └── timofey/
└── ui/
```

## 5. Форматы файлов

| Тип | Формат | Назначение |
|---|---|---|
| Основные UI-изображения | `.webp` | Mini App / Web / Terminal |
| Прозрачные изображения | `.png` | когда нужна прозрачность |
| Исходные фото | `.jpg` / `.heic` / RAW | хранение исходников |
| Документация | `.jpg` / `.png` | скриншоты и инструкции |

Основной рабочий формат интерфейсов: `webp`.

## 6. Размеры изображений

Каждое ключевое изображение готовится в нескольких размерах:

| Variant | Размер | Использование |
|---|---:|---|
| `sm` | `480×480` | mobile / Mini App |
| `md` | `960×960` | tablet / web |
| `lg` | `1600×1600` | terminal / desktop |
| `xl` | `2400×2400` | large terminal / промо |

## 7. Мороженое

### 7.1 Базовое изображение

Базовый продукт:

```text
Ванильное мягкое мороженое в стаканчике
```

Требования:

- только стаканчик, не рожок;
- мягкое мороженое должно быть главным объектом;
- ракурс: 30–45 градусов;
- фон: светлый кремовый или прозрачный;
- освещение: мягкое, без резких теней;
- масштаб стаканчика одинаковый во всех вариантах;
- композиция должна оставлять место для текста в UI.

### 7.2 Запрещено

Не использовать:

- рожки;
- случайную посуду;
- тёмные фоны;
- фотографии с разным масштабом;
- перегруженный фон;
- неестественные цвета;
- изображения, где продукт выглядит меньше второстепенных деталей.

## 8. Сиропы

Для каждого сиропа должно быть два вида изображений:

1. отдельное изображение сиропа;
2. изображение мороженого с этим сиропом.

Примеры:

```text
strawberry.webp
chocolate.webp
caramel.webp
```

Требования:

- единая посуда или единый способ подачи;
- одинаковый ракурс;
- насыщенный, но естественный цвет;
- без лишних предметов;
- визуально понятно, какой это вкус.

## 9. Топпинги

Для каждого топпинга должно быть два вида изображений:

1. отдельное изображение топпинга;
2. изображение мороженого с этим топпингом.

Примеры:

```text
oreo.webp
sprinkles.webp
chocolate_chips.webp
```

Требования:

- крупный план;
- единый фон;
- одинаковый масштаб;
- визуальная читаемость на маленьком экране;
- понятная отличимость одного топпинга от другого.

## 10. Собранные варианты

Для итогового предпросмотра заказа используются готовые изображения комбинаций.

Структура имени:

```text
base_syrup_topping.webp
```

Примеры:

```text
vanilla_strawberry_oreo.webp
vanilla_chocolate_sprinkles.webp
vanilla_caramel_chocolate_chips.webp
```

Правило:

> Если добавляется новый сироп или топпинг, для него создаются все необходимые изображения по текущей продуктовой матрице.

## 11. Тимофей / Клуб Тимоши

Фотографии Тимофея используются как бренд-персонаж.

### 11.1 Роли изображений

```text
welcome.webp        — приветствие
club.webp           — Клуб Тимоши
bonus.webp          — бонусы
gift.webp           — подарок
celebration.webp    — поздравление
profile.webp        — профиль
```

### 11.2 Требования

- естественная улыбка;
- дружелюбное выражение лица;
- светлый фон;
- мягкое освещение;
- безопасный и уважительный образ;
- без лишних персональных деталей;
- одежда в стиле бренда;
- изображение должно вызывать доверие.

### 11.3 Запрещено

Не использовать:

- фотографии плохого качества;
- неудачные эмоции;
- перегруженные фоны;
- фото с посторонними людьми без согласий;
- изображения, где ребёнок выглядит уставшим, расстроенным или вынужденным позировать.

## 12. Именование файлов

Имена файлов пишутся:

- латиницей;
- в нижнем регистре;
- через подчёркивание;
- без пробелов;
- без кириллицы.

Правильно:

```text
vanilla_strawberry_oreo.webp
club_timofey_welcome.webp
syrup_chocolate_sm.webp
```

Неправильно:

```text
Мороженое клубника 1.png
IMG_1234.JPG
final_new_new_2.webp
```

## 13. Метаданные каталога

Каждое изображение должно быть связано с каталогом.

Пример:

```json
{
  "id": "vanilla_strawberry_oreo",
  "base": "vanilla",
  "syrup": "strawberry",
  "topping": "oreo",
  "image": "/images/icecream/composed/vanilla_strawberry_oreo.webp",
  "alt": "Ванильное мягкое мороженое с клубничным сиропом и Oreo"
}
```

## 14. Accessibility

Каждое продуктовое изображение должно иметь `alt`-описание.

Alt должен описывать, что изображено, без рекламного преувеличения.

Пример:

```text
Ванильное мягкое мороженое с шоколадным сиропом и цветной посыпкой
```
