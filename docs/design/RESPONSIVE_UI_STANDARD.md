# Responsive UI Standard

Статус: Draft  
Версия: 0.1  
Проект: Soft ICE Platform / «У Тимоши»

## 1. Назначение

Документ фиксирует стандарт адаптивности интерфейсов Soft ICE Platform.

Все экраны должны корректно работать на разных платформах и устройствах:

- Telegram Mini App;
- MAX Mini App;
- Web App;
- экран вендингового терминала;
- Seller App;
- CRM;
- планшет оператора;
- будущие киоски и панели управления.

## 2. Главный принцип

> Один экран проектируется один раз, но адаптируется под размер, ориентацию и сценарий использования конкретного устройства.

Нельзя проектировать интерфейс только под один экран.

## 3. Классы устройств

| Класс | Пример | Ориентация | Основной сценарий |
|---|---|---|---|
| `mobile` | Telegram / MAX Mini App | portrait | быстрый заказ |
| `tablet` | планшет продавца | portrait / landscape | обслуживание, контроль |
| `desktop` | CRM | landscape | управление и аналитика |
| `terminal` | вендинговый аппарат | portrait / landscape | покупка у автомата |
| `large-terminal` | 32 inch touch screen | portrait / landscape | крупный touch UI |

## 4. Breakpoints

| Token | Ширина | Назначение |
|---|---:|---|
| `breakpoint.mobile` | `0px` | телефоны |
| `breakpoint.mobileLarge` | `390px` | крупные телефоны |
| `breakpoint.tablet` | `768px` | планшеты |
| `breakpoint.desktop` | `1024px` | CRM / desktop |
| `breakpoint.terminal` | `1280px` | терминал |
| `breakpoint.largeTerminal` | `1920px` | большие панели |

## 5. Layout behavior

### 5.1 Mobile / Mini App

- контент в одну колонку;
- нижняя навигация фиксирована;
- крупные карточки;
- минимум текста;
- основные действия в нижней части экрана.

### 5.2 Tablet

- допускается две колонки;
- правая колонка может использоваться для корзины или итогов;
- touch targets не меньше `56px`.

### 5.3 Desktop / CRM

- sidebar navigation;
- таблицы и фильтры;
- карточки могут располагаться сеткой;
- больше плотность информации.

### 5.4 Terminal

- крупные кнопки;
- крупные изображения;
- один главный CTA на экране;
- touch targets не меньше `72px`;
- важные элементы располагаются в зоне комфортного касания.

### 5.5 Large terminal

- изображения масштабируются без потери качества;
- текст увеличивается;
- сетка может быть 2–3 колонки;
- кнопки не должны становиться чрезмерно широкими без необходимости.

## 6. Scaling rules

Компоненты должны иметь варианты:

```text
mobile
tablet
desktop
terminal
large-terminal
```

Размеры не должны быть жёстко зашиты в одном значении.

Правильно:

```text
ProductCard variant="terminal"
Button size="large"
Image mode="cover"
```

Неправильно:

```text
width: 375px
height: 812px
```

## 7. Images

Изображения должны готовиться минимум в трёх размерах:

| Variant | Размер | Назначение |
|---|---:|---|
| `sm` | `480px` | mobile |
| `md` | `960px` | tablet / web |
| `lg` | `1600px` | terminal / desktop |
| `xl` | `2400px` | large terminal / print preview |

Форматы:

- `webp` — основной формат интерфейсов;
- `png` — при необходимости прозрачности;
- `jpg` — только для фото без прозрачности;
- исходники хранятся отдельно.

## 8. Touch targets

| Платформа | Минимум |
|---|---:|
| Mini App | `44px` |
| Seller App | `48px` |
| Tablet | `56px` |
| Terminal | `72px` |
| Large terminal | `88px` |

## 9. Safe areas

Для мобильных приложений учитывать:

- верхнюю системную зону;
- нижнюю системную зону;
- Telegram / MAX WebView;
- экранную клавиатуру;
- вырезы и скругления экрана.

Использовать:

```css
padding-top: env(safe-area-inset-top);
padding-bottom: env(safe-area-inset-bottom);
```

## 10. Orientation

Интерфейсы должны поддерживать:

- portrait;
- landscape — если это требуется устройством.

Для терминала ориентация фиксируется настройками конкретной модели оборудования.

## 11. Content priority

При уменьшении экрана скрываются второстепенные элементы, но не основные действия.

Приоритет:

1. главный CTA;
2. изображение продукта;
3. цена;
4. выбранные параметры;
5. дополнительные подсказки;
6. декоративные элементы.

## 12. Правило тестирования

Каждый экран проверяется минимум на следующих размерах:

```text
360×640
390×844
430×932
768×1024
1024×768
1280×720
1920×1080
1080×1920
```
