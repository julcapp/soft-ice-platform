# Configuration Engine

**Статус:** Draft  
**Версия:** 0.1  
**Проект:** Soft ICE Platform

---

# 1. Назначение

Configuration Engine отвечает за построение допустимой конфигурации любого продукта платформы.

Движок определяет:

- какие варианты доступны пользователю;
- какие комбинации допустимы;
- какие ограничения существуют;
- какую конфигурацию необходимо передать в Pricing Engine, Recipe Engine и Machine Engine.

Configuration Engine не занимается приготовлением продукта и не рассчитывает стоимость.

---

# 2. Место в архитектуре

```
Product
     │
     ▼
Configuration Engine
     │
     ├────────► Pricing Engine
     │
     ├────────► Recipe Engine
     │
     ├────────► Media Engine
     │
     └────────► Machine Engine
```

Configuration Engine является центральным модулем формирования продукта.

---

# 3. Основные задачи

Движок обязан:

- построить конфигурацию продукта;
- проверить совместимость компонентов;
- проверить ограничения рецепта;
- определить итоговый состав продукта;
- передать конфигурацию другим Engine.

---

# 4. Входные данные

Configuration Engine получает:

- Product ID;
- выбранный вкус;
- выбранный размер;
- выбранный сироп;
- выбранный топпинг;
- дополнительные параметры.

---

# 5. Выходные данные

На выходе формируется единый Configuration Object.

Пример:

```json
{
  "productId": "...",
  "flavorId": "...",
  "sizeId": "...",
  "syrupId": "...",
  "toppingId": "...",
  "extras": [],
  "recipeId": "...",
  "mediaId": "..."
}
```

---

# 6. Основные принципы

Configuration Engine:

- не знает о пользовательском интерфейсе;
- не знает о Telegram;
- не знает о CRM;
- не знает о Windows;
- не знает об Android;
- не знает об Ubuntu.

Он работает исключительно с доменной моделью.

---

# 7. Правила

Каждый компонент обязан иметь уникальный ID.

Компоненты должны быть независимыми.

Все проверки выполняются внутри Configuration Engine.

---

# 8. Используемые домены

Configuration Engine взаимодействует со следующими доменами:

- Product
- Flavor
- Size
- Syrup
- Topping
- Recipe
- Media
- Pricing
- Machine

---

# 9. Не входит в обязанности

Configuration Engine не:

- рассчитывает цену;
- запускает приготовление;
- начисляет бонусы;
- принимает оплату;
- отображает интерфейс.

---

# 10. Архитектурный принцип

Configuration Engine является единственным источником истины о составе продукта.

Все остальные подсистемы получают готовую конфигурацию исключительно через него.

---

# 11. Долгосрочная цель

Движок должен поддерживать не только мягкое мороженое.

Архитектура должна позволять конфигурировать:

- мягкое мороженое;
- молочные коктейли;
- кофе;
- лимонады;
- десерты;
- горячие напитки;
- любые будущие продукты платформы.

---
