# YooKassa Verification

Статус: Draft  
Ответственный блок: Payments / Telegram Bot / Mini App  
Проект: Soft ICE Platform / «У Тимоши»

## 1. Цель документа

Документ фиксирует требования ЮKassa к проверке Telegram-бота `ice_robo_Bot` (`@desserty_bot`) и описывает комплект материалов, который необходимо подготовить для повторной проверки.

## 2. Последний запрос ЮKassa

ЮKassa указала следующие требования:

> Вы поменяли название бота, нужны новые скриншоты (отдельные скриншоты, не одной картинкой).
>
> Пожалуйста, пришлите нам полноэкранные необрезанные скриншоты (до и после перехода в мини-приложение): должно быть видно название бота, карточки товаров/услуг с ценами в рублях и описанием (несколько примеров).
>
> Если для наполнения бота вам нужна интеграция, вы можете воспользоваться тестовым магазином.

## 3. Что должно быть видно на скриншотах

Обязательные элементы:

1. Актуальное название бота: `ice_robo_Bot`.
2. Username бота: `@desserty_bot`.
3. Полноэкранный интерфейс Telegram без обрезки.
4. Экран до перехода в Mini App.
5. Экран после перехода в Mini App.
6. Карточки товаров/услуг.
7. Цены в рублях.
8. Описание товаров/услуг.
9. Несколько примеров позиций.
10. Отдельные файлы скриншотов, не коллаж.

## 4. Позиции для демонстрации

Для проверки используются три позиции.

### 4.1 Мороженое «Вкус дня»

Тип: товар  
Цена: 130 ₽

Описание:

Стаканчик мягкого мороженого. В стоимость входит один сироп и один топпинг на выбор.

Кнопка: `Купить`

### 4.2 Клуб Тимоши

Тип: услуга / пополнение клубного счёта  
Цена: 300 ₽

Описание:

Пополнение клубного счёта. После пополнения гость получает доступ к скидке 20%, бонусной программе, специальным предложениям и акциям.

Кнопка: `Пополнить`

### 4.3 Подарочный сертификат

Тип: товар / цифровой сертификат  
Цена: 500 ₽

Описание:

Электронный подарочный сертификат для покупки мягкого мороженого «У Тимоши».

Кнопка: `Купить сертификат`

## 5. Требуемый сценарий Telegram-бота

### Шаг 1. Открытие бота

Проверяющий открывает Telegram-бота.

Должно быть видно:

- `ice_robo_Bot`;
- `@desserty_bot`;
- кнопка `START`.

### Шаг 2. Нажатие START

Бот показывает приветствие:

```text
🍦 Добро пожаловать в «У Тимоши»!

Мы готовим свежее мягкое мороженое и дарим бонусы постоянным гостям.

Выберите действие.
```

Кнопки:

```text
🍦 Купить мороженое
🎁 Клуб Тимоши
🎫 Подарочный сертификат
📱 Открыть Mini App
📍 Где купить
❓ Помощь
```

### Шаг 3. Карточка мороженого

При нажатии `🍦 Купить мороженое` бот показывает карточку:

```text
🍦 Мороженое «Вкус дня»

Стаканчик мягкого мороженого.

В стоимость входит:
✓ 1 сироп;
✓ 1 топпинг.

Цена: 130 ₽
```

Кнопка:

```text
Купить
```

### Шаг 4. Карточка клуба

При нажатии `🎁 Клуб Тимоши` бот показывает карточку:

```text
🎁 Клуб Тимоши

Пополнение клубного счёта.

После пополнения вы получаете:
✓ скидку 20%;
✓ бонусную программу;
✓ специальные предложения;
✓ участие в акциях.

Стоимость: 300 ₽
```

Кнопка:

```text
Пополнить
```

### Шаг 5. Карточка сертификата

При нажатии `🎫 Подарочный сертификат` бот показывает карточку:

```text
🎫 Подарочный сертификат

Электронный сертификат для покупки мягкого мороженого «У Тимоши».

Номинал: 500 ₽
```

Кнопка:

```text
Купить сертификат
```

### Шаг 6. Переход в Mini App

При нажатии `📱 Открыть Mini App` Telegram открывает Mini App по адресу:

```text
https://app.utimoshi.ru
```

## 6. Требуемый комплект скриншотов

Скриншоты отправляются отдельными файлами.

| № | Содержание скриншота | Обязательные признаки |
|---:|---|---|
| 1 | Открытый бот | Видно `ice_robo_Bot`, `@desserty_bot`, кнопка START |
| 2 | Приветствие после START | Видно меню бота |
| 3 | Карточка «Мороженое Вкус дня» | Видно цену 130 ₽ и описание |
| 4 | Карточка «Клуб Тимоши» | Видно цену 300 ₽ и описание |
| 5 | Карточка «Подарочный сертификат» | Видно цену 500 ₽ и описание |
| 6 | Экран с кнопкой Mini App | Видно кнопку `📱 Открыть Mini App` |
| 7 | Переход в Mini App | Видно системный экран/открытие Telegram Mini App |
| 8 | Mini App открыт | Видно интерфейс Mini App после перехода |

## 7. Требования к скриншотам

1. Не делать коллаж.
2. Не обрезать верхнюю часть Telegram.
3. Не скрывать название бота.
4. Не скрывать username.
5. На карточках должны быть цены именно в рублях.
6. Описание товара/услуги должно быть читаемым.
7. Скриншоты должны быть сделаны из реального Telegram, а не из макета.

## 8. Тестовый магазин

ЮKassa предлагает использовать тестовый магазин для наполнения бота.

На текущем этапе тестовый магазин не является обязательным, если бот показывает карточки товаров/услуг с описанием, ценой и переходом в Mini App.

Если ЮKassa дополнительно запросит подтверждение тестовой оплаты, будет подключён тестовый магазин ЮKassa.
